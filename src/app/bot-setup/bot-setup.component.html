Creating Your Bot Application and Connecting it to Your Instance

Now that you are able to run the code, you need to connect the code to a Discord Bot application. If you have not created a Discord Bot application, you will need to follow these steps:

  • Log in to the Discord Developer site
  • Create a new application by clicking on the "New Application" button.
  • On the left, select "Bot", then create a new bot.

Create a Webhook (Optional)

The bot is configured to send an online notice, and updates on servers it leaves or joins, through a Discord Webhook. In your server, create a new Webhook and copy the URL. You will need the ID and Token from that URL for the next step.

Link your Bot to the Code

Now that you have a bot, you'll need to create a file called .env. This file will contain the following variables:

  • DISCORD_TOKEN - (Required) - The value of this variable will be your Bot Token from the Discord developer page. It is VERY important that you keep this token a secret - do not upload it to GitHub or share it publically, as this will allow anyone to access your discord bot.
  • HABITICA_KEY - (Optional for habitica command) - This is your API key for Habitica.
  • HP_KEY - (Optional for hp commands) - This is the API key for the Harry Potter API.
  • MONGODB - (Required) - The value of this variable will be your MongoDB authentication URI. Like the bot token, this needs to be kept secret to prevent unauthorised access to and modification of your database.
  • NASA_API - (Optional for space command) - This is the API key for the NASA API.
  • OWNER_ID - (Optional) - This tells the bot which Discord account is the owner. The bot will respond just a little differently to the owner. Leaving this blank WILL lock out some commands.
  • PRODDEV - (Optional) - This tells the bot whether it is in a development state or production state - the webhook message includes this value.
  • WH_ID - (Optional) - This is the ID for the Discord webhook.
  • WH_TOKEN - (Optional) - This is the token for the Discord webhook.
